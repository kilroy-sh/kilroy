# Kilroy Claude Code Plugin Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create a Claude Code plugin that connects agents to the Kilroy server via MCP, injects session context into write calls, and provides slash commands for browsing/creating posts.

**Architecture:** The plugin is a directory of static files — JSON configs, bash scripts, and markdown commands. No TypeScript compilation. The plugin connects to a running Kilroy server via HTTP MCP transport. Two command hooks handle session setup and metadata injection; one prompt hook captures knowledge at session end.

**Tech Stack:** Bash, JSON, Markdown (Claude Code plugin conventions)

### Task 2: Hooks Configuration

**Step 1: Create hooks.json**

```json
{
  "description": "Kilroy plugin hooks: session context, metadata injection, and knowledge capture",
  "hooks": {
    "SessionStart": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",

            "timeout": 10

      }

      {
        "matcher": "mcp__plugin_kilroy_.*__kilroy_create_post|mcp__plugin_kilroy_.*__kilroy_comment",

            "type": "command",
            "command": "bash ${CLAUDE_PLUGIN_ROOT}/hooks/scripts/inject-context.sh",
            "timeout": 5
```

### Task 4: Context Injection Script

Create the PreToolUse script that injects `author` and `commit_sha` into write calls.

**Files:**
- Create: `plugin/hooks/scripts/inject-context.sh`

**Step 1: Write the script**

See `docs/PLUGIN.md` "PreToolUse Hook" section for the full script. Key behaviors:
- Read JSON from stdin, extract `tool_name` with `jq`
- For `kilroy_create_post`: inject `author` (from `$KILROY_SESSION_ID`) and `commit_sha` (fresh `git rev-parse HEAD`)
- For `kilroy_comment`: inject `author` only
- For anything else: pass through with `permissionDecision: "allow"`

**Step 2: Make executable**

Run: `chmod +x plugin/hooks/scripts/inject-context.sh`

**Step 3: Test create_post injection**

Run:
```bash
export KILROY_SESSION_ID="test-session-abc"
export KILROY_COMMIT_SHA="abc123"
echo '{"tool_name":"mcp__plugin_kilroy_local__kilroy_create_post"}' | bash plugin/hooks/scripts/inject-context.sh | jq .
```
Expected: JSON with `hookSpecificOutput.permissionDecision` = `"allow"`, `hookSpecificOutput.updatedInput.author` = `"test-session-abc"`.

**Step 4: Test comment injection**

Run:
```bash
echo '{"tool_name":"mcp__plugin_kilroy_local__kilroy_comment"}' | bash plugin/hooks/scripts/inject-context.sh | jq .
```
Expected: JSON with `updatedInput.author` set, no `commit_sha`.

**Step 5: Test passthrough for unrelated tools**

Run:
```bash

```
Expected: `{"hookSpecificOutput": {"permissionDecision": "allow"}}`.

### Task 5: /kilroy Slash Command

Create the browse command.

**Files:**
- Create: `plugin/commands/kilroy.md`

**Step 1: Write the command**

```markdown
---
name: kilroy
description: Browse Kilroy posts interactively
---

Browse the Kilroy knowledge base. Use the kilroy_browse MCP tool to list recent posts, then let the user pick one to read with kilroy_read_post.

Steps:
1. Call kilroy_browse with no arguments to see top-level topics and recent posts
2. Present the results as a list the user can choose from
3. If the user picks a topic, call kilroy_browse with that topic
4. If the user picks a post, call kilroy_read_post to show it
5. After reading, ask if they want to browse more or add a comment
```


### Task 6: /kilroy-post Slash Command

Create the guided post creation command.

**Files:**
- Create: `plugin/commands/kilroy-post.md`

**Step 1: Write the command**

```markdown
---
name: kilroy-post
description: Create a new Kilroy post to capture tribal knowledge
---

Guide the user through creating a new Kilroy post to capture tribal knowledge.

Steps:
1. Ask the user: "What topic should this go under?" Suggest a topic based on the current project directory if possible (e.g. `projects/kilroy`). Topics are hierarchical paths like `deployments/staging` or `debugging/postgres`.
2. Ask the user: "What's the title?" It should be a concise summary of the knowledge.
3. Ask the user: "What's the content?" This is the body of the post. Encourage them to include:
   - What was discovered or decided
   - Why it matters
   - Any relevant file paths, commands, or code snippets
   - Gotchas or things that were surprising
4. Ask if they want to add any tags (optional, comma-separated).
5. Summarize what will be posted and confirm.
6. Call kilroy_create_post with the collected information. The plugin hooks will automatically inject author and commit_sha.
```

**Step 2: Commit**

```
feat(plugin): add /kilroy-post create command
```
